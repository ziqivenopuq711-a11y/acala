Karura — Canary Network

Karura is the canary network of Acala, deployed as parachain 2000 on the Kusama relay chain. "Canary network" means Karura operates with real economic value but is used to test and validate new features before they are deployed on Acala mainnet. Kusama itself is Polkadot's canary chain, known for faster governance and a higher risk tolerance.

Karura has its own native token KAR (as opposed to ACA on Acala mainnet), its own stablecoin kUSD, and liquid staking token LKSM (Liquid KSM). The SS58 prefix for Karura addresses is 8.

Migration & Upgrade Notes

When migrating applications between Acala runtime upgrades or moving from Karura to Acala mainnet, keep the following in mind:

  • Re-encode SS58 addresses: Karura uses prefix 8 and Acala uses prefix 10. Always re-encode addresses when switching networks. The underlying public key is identical.
  • Refresh metadata: After a runtime upgrade (specVersion bump), call state_getMetadata to refresh your local type registry before constructing extrinsics.
  • Recalibrate fees: Transaction weight and fee coefficients may change between spec versions. Always use payment_queryInfo to estimate fees dynamically rather than hardcoding values.
  • Check transaction version: The current transactionVersion is 3. Signed extrinsics include this value; mismatches will cause rejection.
  • Verify genesis hash: All signed transactions include the genesis hash. Ensure your signing library is configured with 0xfc41b9bd8ef8fe53d58c7ea67c794c7ec9a73daf05e6d54b14ff6342c99ba64c.
