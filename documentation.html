What is Acala?

Acala is a Substrate-based blockchain that operates as parachain ID 2000 on the Polkadot relay chain. It is designed specifically as the DeFi hub of the Polkadot ecosystem, providing core financial infrastructure that other parachains can compose with via XCM cross-chain messaging.

The network launched its mainnet in December 2021 after winning a Polkadot parachain slot auction. Its native governance and utility token is ACA, used for transaction fees, governance voting, and incentivising liquidity providers. The SS58 address prefix for Acala is 10, meaning all Acala addresses begin with a distinctive prefix that differentiates them from other Substrate chains.

Karura is Acala's canary network, deployed as parachain 2000 on the Kusama relay chain. Karura carries its own native token KAR and acts as a real-value, fast-moving testing ground for features before they graduate to Acala mainnet. Both networks share the same codebase and architecture.

Key Concepts

aUSD Stablecoin

aUSD is Acala's decentralised, over-collateralised stablecoin soft-pegged to the US dollar. Users lock accepted collateral assets (such as DOT, LDOT, or LP tokens) into Collateral Debt Positions (CDPs) to mint aUSD. A stability fee is charged on outstanding debt, and positions are liquidated if the collateralisation ratio falls below the required minimum. aUSD can be transferred via XCM to other Polkadot parachains, serving as a cross-chain stablecoin primitive.

LDOT Liquid Staking

LDOT (Liquid DOT) is Acala's liquid staking derivative. Users deposit DOT into the Acala liquid staking protocol and receive LDOT in return, representing their staked DOT plus accumulated staking rewards. LDOT is freely tradeable and can be used as collateral for aUSD or provided as liquidity in the DEX, solving the opportunity cost problem of traditional staking where assets are locked and illiquid.

Built-in DEX

Acala includes a native automated market maker (AMM) DEX as a Substrate pallet, not a smart contract. This means the DEX benefits from the same security and performance as the base layer. Users can create liquidity pools, provide liquidity to earn fees, and swap tokens trustlessly. The DEX is deeply integrated with aUSD and LDOT, enabling complex DeFi interactions like collateralise-and-swap in a single transaction.

XCM Cross-Chain Messaging

XCM (Cross-Consensus Messaging) is Polkadot's standard for cross-chain communication. Acala uses XCM to send and receive assets and messages from other parachains without relying on a bridge. For example, DOT can be teleported from the relay chain to Acala for use as collateral, and aUSD can be sent from Acala to a DEX on another parachain — all atomically and trustlessly.

EVM+ Layer

Acala's EVM+ is an Ethereum Virtual Machine implementation embedded within the Substrate runtime. It goes beyond standard EVM compatibility by allowing EVM smart contracts to directly call Substrate pallets and vice versa. This means Solidity contracts on Acala can interact with the DEX, aUSD, and liquid staking natively. Developers can use familiar tools: Hardhat, Truffle, ethers.js, and MetaMask.

ACA Token

ACA is the native token of Acala Network with a fixed maximum supply. It serves multiple purposes: paying transaction fees (with a unique fee payment flexibility allowing fees in other tokens), participating in on-chain governance (council elections, referenda), and serving as a system reserve for the aUSD stability mechanism. ACA holders can stake to earn a share of network revenue.

Quick Start

The fastest way to connect to Acala is via the Dwellir WebSocket endpoint. You will need an API key from dashboard.dwellir.com.

  1. Sign up at dashboard.dwellir.com and create an Acala endpoint to receive your API key.
  2. Install the Polkadot.js API: npm install @polkadot/api
  3. Connect using WebSocket: wss://api-acala.n.dwellir.com/YOUR_API_KEY
  4. Query the chain: await api.rpc.system.chain() should return Acala
