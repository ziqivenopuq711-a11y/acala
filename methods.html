RPC Methods

Complete reference for Substrate JSON-RPC methods available on Acala Network via Dwellir endpoints.

Method Reference

Method Namespace Description
system_health system Returns node health: peer count, sync status, and whether the node should be considered healthy.
system_properties system Returns chain properties including SS58 prefix (10), token decimals (12), and token symbol (ACA).
system_chain system Returns the name of the current chain. For Acala mainnet this returns the string Acala.
system_accountNextIndex system Returns the next valid nonce for a given account address, accounting for pending transactions in the mempool.
chain_getHeader chain Returns the block header for a given block hash. Omit the hash parameter to get the latest block header.
chain_getBlock chain Returns the full block (header + extrinsics) for a given block hash.
chain_getBlockHash chain Returns the block hash for a given block number. Pass 0 to get the genesis hash.
chain_getFinalizedHead chain Returns the hash of the latest finalised block. Use this as an anchor for queries that require finality.
state_getRuntimeVersion state Returns the runtime version object including specVersion (2300) and transactionVersion (3).
state_getStorage state Returns the raw SCALE-encoded storage value at a given storage key, optionally at a given block hash.
state_getKeysPaged state Returns a paginated list of storage keys with a given prefix. Use for iterating over large storage maps without timeouts.
state_getMetadata state Returns the full SCALE-encoded runtime metadata, describing all pallets, storage items, calls, and events.
author_submitExtrinsic author Submits a signed or unsigned extrinsic to the node's transaction pool. Returns the extrinsic hash.
payment_queryInfo payment Returns fee estimation information (partial fee, weight, class) for a given encoded extrinsic at an optional block hash.

cURL Examples

system_health

curl -H "Content-Type: application/json" \
  -d '{"id":1,"jsonrpc":"2.0","method":"system_health","params":[]}' \
  https://api-acala.n.dwellir.com/YOUR_API_KEY

# Expected response:
# {"jsonrpc":"2.0","result":{"isSyncing":false,"peers":25,"shouldHavePeers":true},"id":1}

  -d '{"id":1,"jsonrpc":"2.0","method":"chain_getHeader","params":[]}' \
  https://api-acala.n.dwellir.com/YOUR_API_KEY

# Returns block number, parent hash, state root, extrinsics root, and digest

TypeScript Examples

system_properties & chain_getHeader

import { ApiPromise, WsProvider } from '@polkadot/api';

const api = await ApiPromise.create({
  provider: new WsProvider('wss://api-acala.n.dwellir.com/YOUR_API_KEY')
});

// system_properties
const props = await api.rpc.system.properties();
console.log('SS58 prefix:', props.ss58Format.unwrap().toNumber()); // 10
console.log('Token symbol:', props.tokenSymbol.unwrap()[0].toString()); // ACA
console.log('Token decimals:', props.tokenDecimals.unwrap()[0].toNumber()); // 12

// chain_getHeader
const header = await api.rpc.chain.getHeader();
console.log('Block number:', header.number.toNumber());
console.log('State root:', header.stateRoot.toHex());

payment_queryInfo & author_submitExtrinsic

// Build an extrinsic
const tx = api.tx.balances.transferKeepAlive(
  '23M5ttkmR6Kco6wSSGtxKDM5gbeXqAXbBRdCy4m2VHNnJPFP',
  1_000_000_000_000n // 1 ACA
);

// payment_queryInfo — estimate fees
const paymentInfo = await api.rpc.payment.queryInfo(tx.toHex());
console.log('Estimated fee:', paymentInfo.partialFee.toHuman());

// author_submitExtrinsic — sign and send
const { nonce } = await api.query.system.account(account.address);
const signedTx = await tx.signAsync(account, { nonce });
const txHash = await api.rpc.author.submitExtrinsic(signedTx);
console.log('Submitted tx hash:', txHash.toHex());

Smoke Test Checklist

Run this sequence to verify your connection is healthy and the node is fully synced before building production integrations:

  1. system_health — confirm isSyncing: false and peers > 5. A syncing node will return stale data.
  2. chain_getHeader — verify the returned block number is recent (check against Subscan). A stale block number indicates the node is behind.
  3. chain_getFinalizedHead — confirm finalized head is progressing. On Polkadot, finality typically lags the best head by 1–2 blocks. A large gap suggests a relay chain issue.
  4. state_getRuntimeVersion — verify specVersion: 2300 and transactionVersion: 3. If these differ, update your signing library and type bundle accordingly. 
// Quick smoke test script
const checks = await Promise.all([
  api.rpc.system.health(),
  api.rpc.chain.getHeader(),
  api.rpc.chain.getFinalizedHead(),
  api.rpc.state.getRuntimeVersion(),
]);

const [health, header, finalized, version] = checks;

console.assert(!health.isSyncing.isTrue, 'Node is still syncing!');
console.assert(health.peers.toNumber() > 5, 'Low peer count!');
console.assert(version.specVersion.toNumber() === 2300, 'Unexpected spec version!');
console.assert(version.transactionVersion.toNumber() === 3, 'Unexpected tx version!');
console.log('All smoke tests passed ✓');
