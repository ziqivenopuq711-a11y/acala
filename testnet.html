Mandala Testnet

Acala's public testnet for developers to test DeFi integrations before mainnet deployment.

Overview

Mandala is Acala's public test network, designed to closely mirror the Acala mainnet environment. It runs the same Substrate runtime, pallets, and EVM+ configuration as mainnet, allowing developers to test transactions, smart contracts, and DeFi integrations without risking real assets.

Mandala uses test tokens with no real monetary value. These tokens can be obtained for free from the Acala faucet. The testnet is periodically reset to clear state, so do not rely on it for persistent data. Always test on Mandala before deploying to mainnet or Karura.

Mandala is also used by the Acala core team to validate runtime upgrades, new pallet introductions, and governance changes before they are applied to Karura or Acala mainnet.

Getting Test Tokens

Mandala testnet tokens can be obtained free of charge via the Acala faucet. These tokens have no real monetary value and exist solely for testing purposes.

  1. Join the Acala Discord server and navigate to the #testnet-faucet channel.
  2. Send the command: !drip YOUR_ACALA_ADDRESS where the address uses SS58 prefix 10.
  3. The bot will send test ACA, aUSD, and DOT tokens to your address within seconds.
  4. Verify receipt using Polkadot.js Apps connected to the Mandala WSS endpoint, or via the Blockscout explorer.
Note: Faucet tokens are rate-limited. Each address can request tokens once every 24 hours. If you need large amounts for stress testing, contact the Acala team directly.

Connecting Your Wallet

Polkadot.js Extension

Install the Polkadot.js browser extension to manage Substrate accounts. Then connect Polkadot.js Apps to the Mandala WebSocket endpoint.

  1. Go to polkadot.js.org/apps
  2. Click the network selector (top-left)
  3. Select "Live Networks" → custom endpoint
  4. Enter: wss://mandala-tc9.aca-staging.network
  5. Click "Switch" and wait for connection

MetaMask (EVM+)

Add Mandala as a custom network in MetaMask to test EVM+ contracts and transactions.

Testing Workflow Best Practices

  • Start with unit tests using a local Mandala node (via Docker or cargo run) before using the public testnet.
  • Use the public Mandala testnet for integration testing of multi-pallet or cross-chain workflows.
  • Verify on Karura with real (but lower-value) assets before Acala mainnet deployment — Karura has a more aggressive runtime upgrade cadence.
  • Test all edge cases: insufficient collateral, slippage in DEX swaps, failed XCM transfers, and extrinsic weight limits.
  • Check event emissions: subscribe to system.events to confirm expected pallet events fire correctly.
  • Simulate runtime upgrades on a local fork of Mandala before submitting governance proposals on mainnet.
  • Never use mainnet keys on testnet or vice versa — keep separate key pairs to avoid accidental exposure of mainnet private keys.
