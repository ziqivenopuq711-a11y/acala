RPC Endpoints

Dwellir-managed RPC infrastructure for Acala Network — reliable, scalable, and always available.

Available Endpoints

🌐

HTTPS Endpoint

Use for single, stateless requests: querying storage, submitting pre-signed extrinsics, and polling chain state. Each request is an independent HTTP POST to the JSON-RPC endpoint. 

https://api-acala.n.dwellir.com/YOUR_API_KEY
🔌

WebSocket Endpoint

Preferred for subscriptions, long-running sessions, and real-time data. Supports state_subscribeStorage, chain_subscribeNewHeads, and all pub/sub methods. 

wss://api-acala.n.dwellir.com/YOUR_API_KEY
Note: Replace YOUR_API_KEY with your personal API key. Keys are available from the Dwellir Dashboard after sign-up. Each key is rate-limited per tier; contact support@dwellir.com to discuss higher limits.

API Key Authentication

Dwellir uses API key authentication embedded in the URL path. There are no custom headers required — simply append your key as the last path segment of the endpoint URL.

  1. Navigate to dashboard.dwellir.com and sign in or create an account.
  2. Click Create New Endpoint and select Acala from the network list.
  3. Your API key will be displayed in the endpoint URL, e.g., abc123def456.
  4. Use the full URL in your application — the key is authenticated server-side with each request.
  5. Monitor usage, rotate keys, and set rate limit alerts from the dashboard at any time.

JavaScript / @polkadot/api

const provider = new WsProvider(
  'wss://api-acala.n.dwellir.com/abc123def456'
);

Python / substrate-interface

substrate = SubstrateInterface(
  url="wss://api-acala.n.dwellir.com/abc123"
)

WebSocket vs HTTPS

Feature WebSocket (WSS) HTTPS
Connection Persistent, full-duplex Request/response, stateless
Subscriptions ✓ Supported ✗ Not supported
Real-time Events ✓ Push-based Requires polling
Best For Wallets, dApps, indexers, bots Scripts, one-off queries, CI checks
Overhead Low after handshake TLS handshake per batch

Best Practices

Cache Metadata & Types

Runtime metadata (fetched via state_getMetadata) is large and rarely changes. Cache it locally and only refresh when the specVersion changes. Similarly, cache type bundle definitions to avoid redundant network calls on startup.

Paginated Storage Scans

Use state_getKeysPaged instead of state_getKeys for large storage maps. The paged version accepts a count and startKey argument, preventing timeouts and oversized responses.

Reconnection & Backoff

Implement exponential backoff when WebSocket connections drop. The @polkadot/api library handles basic reconnection automatically, but you should handle the error event and re-subscribe to any active subscriptions after reconnecting.

Connection Pooling

For server-side applications handling many concurrent users, maintain a pool of WebSocket connections rather than opening a new connection per request. Share a single ApiPromise instance across your application and use it concurrently — it is designed for concurrent access.

Troubleshooting

Connection Refused / 401 Unauthorized
Verify your API key is correct and that your Dwellir plan is active. Check the dashboard for quota exhaustion. Ensure you are not appending extra path segments after the key.
Invalid SS58 Address Error
Acala uses SS58 prefix 10. If you see "Invalid SS58 address", re-encode the address with prefix 10 using encodeAddress(decodeAddress(address), 10) from @polkadot/util-crypto.
Type Decoding Errors
After a runtime upgrade, cached type definitions may be stale. Call state_getMetadata to retrieve fresh metadata and reinitialise your API instance. Ensure you are using a version of @polkadot/api compatible with spec version 2300.
Extrinsic Failures (1010 / Bad Signature)
Verify that the genesis hash in your signed payload matches 0xfc41b9bd...ba64c and that the transactionVersion is set to 3. Also confirm the account nonce is current — fetch it fresh with system_accountNextIndex.
