Running a Node

Set up a collator, archive, or full node to participate in the Acala network infrastructure.

Node Types

🏗 Collator Node

Collators are responsible for building parachain blocks and submitting them to the Polkadot relay chain validators for inclusion and finality. They collect transactions from the mempool, execute the runtime, and produce Proof-of-Validity (PoV) blocks.

Running a collator requires bonding ACA and being selected via the collator selection pallet. Collators earn a portion of transaction fees as rewards.

📚 Archive Node

Archive nodes store the complete historical state of the blockchain — every block, every storage entry at every block height. This makes them essential for explorers, indexers (SubQuery, The Graph), and applications that need to query historical state.

Archive nodes require significantly more disk space (several TB for long-running chains) but provide the full state_getStorage history.

⚙️ Full Node

Full nodes synchronise the entire blockchain and validate all blocks but prune older state to save disk space. They serve RPC requests for current state but cannot answer historical state queries beyond the pruning window (typically 256–1024 blocks).

Full nodes are suitable for running your own private RPC endpoint, participating in the p2p network, and broadcasting transactions.

Installation

Option 1: Pre-built Binary

Download the latest pre-built Acala node binary from the official GitHub releases page. This is the fastest way to get started. 

# Download the latest binary
wget https://github.com/AcalaNetwork/Acala/releases/latest/download/acala

# Make executable
chmod +x ./acala

# Verify binary
./acala --version

Option 2: Build from Source

Building from source requires Rust and the Substrate build dependencies. Ensure you have at least 16 GB RAM for compilation. 

# Install Rust
curl https://sh.rustup.rs -sSf | sh
rustup default stable
rustup update nightly
rustup target add wasm32-unknown-unknown --toolchain nightly

# Install system dependencies (Ubuntu)
sudo apt install -y cmake pkg-config libssl-dev git clang libclang-dev

# Clone and build
git clone https://github.com/AcalaNetwork/Acala.git
cd Acala
cargo build --release -p acala-node

# Binary location
./target/release/acala --version

Starting a Full Node

# Start a full node syncing Acala mainnet
./acala \
  --chain=acala \
  --name="my-acala-node" \
  --rpc-port=9944 \
  --ws-port=9945 \
  --rpc-cors=all \
  --rpc-methods=safe \
  --base-path=/data/acala \
  -- \
  --chain=polkadot

# Archive node (add --state-pruning=archive --blocks-pruning=archive)
./acala \
  --chain=acala \
  --state-pruning=archive \
  --blocks-pruning=archive \
  --base-path=/data/acala-archive \
  -- \
  --chain=polkadot

Configuration & Monitoring

Acala nodes expose a Prometheus metrics endpoint (default: port 9615) that can be scraped by Prometheus and visualised in Grafana. Key metrics to monitor include:

  • substrate_block_height — ensure the node is keeping up with the best and finalized head
  • substrate_sub_libp2p_peers_count — peer count should remain above 10
  • substrate_tasks_polling_duration — high values indicate CPU saturation
  • Disk IOPS and available space — NVMe SSDs are strongly recommended for write-intensive archive nodes

Enable metrics with: --prometheus-external --prometheus-port 9615
